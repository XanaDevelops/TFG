## Brief overview
This rule file defines the commenting style preferences for the TFG project. Comments should be minimal and high-signal, avoiding unnecessary explanations.

## Commenting guidelines
- Do not write changelogs or play-by-play descriptions of code changes (e.g., "removed foo", "doing Z")
- Do not state the obvious or explain self-explanatory code structure
- Write comments only when absolutely necessary to explain why a non-standard or complex approach was taken
- Keep all comments under one line
